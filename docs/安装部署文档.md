# 安装部署文档

## 1. 适用场景

本文档用于将 `calendar-sync` 部署到 Linux 服务器，长期运行“钉钉日历 -> iCloud 日历”的单向同步服务。

## 2. 环境要求

1. 操作系统：Linux（Ubuntu / Debian / CentOS 均可）。
2. Node.js：`>= 20`。
3. npm：`>= 10`。
4. 网络：服务器可访问钉钉 CalDAV 与 iCloud CalDAV。
5. 凭据：
   - 钉钉 CalDAV：`baseUrl / username / password`。
   - iCloud：`Apple ID + App 专用密码`。

## 3. 安装步骤

1. 上传项目到服务器目录，例如：

```bash
cd /opt
mkdir -p calendar-sync
cd calendar-sync
```

2. 放置项目文件后，执行：

```bash
npm install
```

3. 初始化配置：

```bash
cp .env.example .env
```

4. 编辑 `.env`，至少填写：

```dotenv
DINGTALK_CALDAV_BASE_URL=...
DINGTALK_CALDAV_USERNAME=...
DINGTALK_CALDAV_PASSWORD=...

ICLOUD_CALDAV_BASE_URL=https://caldav.icloud.com
ICLOUD_APPLE_ID=...
ICLOUD_APP_SPECIFIC_PASSWORD=...
```

可选但推荐：

```dotenv
# 钉钉来源日历（可多选，CSV；为空=全部）
DINGTALK_CALDAV_SOURCE_CALENDAR_NAMES=
DINGTALK_CALDAV_SOURCE_CALENDAR_HREFS=
# iCloud 目标日历（单选；为空=自动选择可写日历）
ICLOUD_TARGET_CALENDAR_NAME=
ICLOUD_TARGET_CALENDAR_HREF=
```

说明：服务正式运行只读取当前目录 `.env`，不再从 `test/` 目录读取环境变量。

## 4. 首次验收（强烈建议）

1. 验证钉钉可读：

```bash
npm run test:dingtalk:read
```

2. 验证 iCloud 可读写：

```bash
npm run test:icloud:rw
```

3. DryRun 验证同步行为：

```bash
SYNC_DRY_RUN=true npm run sync:once
```

4. 真正执行一次同步：

```bash
SYNC_DRY_RUN=false npm run sync:once
```

## 5. 生产参数建议

建议写入 `.env`：

```dotenv
SYNC_DRY_RUN=false
SYNC_ENABLE_DELETE=true
SYNC_DELETE_CONFIRM_RUNS=2
SYNC_DELETE_MAX_RATIO=0.9
SYNC_INTERVAL_SECONDS=300
HEARTBEAT_INTERVAL_SECONDS=30
SYNC_RUN_ON_STARTUP=true

LOG_FILE=/opt/calendar-sync/data/logs/calendar-sync.log
LOG_LEVEL=info

HOST=0.0.0.0
PORT=8787
```

## 6. 启动方式

### 6.1 前台启动（调试）

```bash
npm start
```

### 6.2 生产参数快捷启动

```bash
npm run start:prod
```

## 7. 服务验证

1. 健康检查：

```bash
curl -s http://127.0.0.1:8787/health
```

2. 状态检查：

```bash
curl -s http://127.0.0.1:8787/api/status
```

3. 查看日志：

```bash
curl -s "http://127.0.0.1:8787/api/logs?lines=50"
```

4. 手动触发同步：

```bash
curl -s -X POST http://127.0.0.1:8787/api/sync
```

若配置了 `SYNC_API_TOKEN`：

```bash
curl -s -X POST -H "Authorization: Bearer $SYNC_API_TOKEN" http://127.0.0.1:8787/api/sync
```

5. 打开 Web 控制台：

```text
http://<服务器IP>:8787/
```

## 8. CLI 运维命令

```bash
# 状态
npm run cli -- status --json

# 日志
npm run cli -- logs --lines 200

# 手动单次同步
npm run cli -- sync once --dry-run false --enable-delete true
```

## 9. 常见问题

1. 未配置 `ICLOUD_TARGET_CALENDAR_NAME/HREF` 是否会失败？
   - 不会。系统会自动选择第一个可写 iCloud 日历（会尽量避开“提醒/节假日”等只读倾向日历）。
2. 同步删除不执行
   - 检查 `SYNC_ENABLE_DELETE=true`。
   - 检查 `SYNC_DELETE_CONFIRM_RUNS` 是否达到连续轮次。
   - 检查是否触发 `SYNC_DELETE_MAX_RATIO` 删除保护。
3. `status` 显示离线
   - 说明服务未启动，CLI 已自动降级为读取本地状态文件。

## 10. 升级建议

1. 升级前备份 `data/sync-state.json` 与日志目录。
2. 先执行 `SYNC_DRY_RUN=true npm run sync:once` 观察行为。
3. 验证无异常后再恢复 `SYNC_DRY_RUN=false`。
